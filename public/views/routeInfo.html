General info

This application is meant to be used in "machine-to-machine" way. Meaning, while you can manually search and insert data, to really benefit from it, you have to code dynamic HTTP REST requests into your game.

Below you can see all the routes that this API implements. You can click the route links to see how everything works and is formatted, just remember to replace the apikey with your own. Each object that gets queried from the server is appended with links to relating data. This is called HATEOS and it is a self documenting system.

For example, to POST game data, you would use the following URL: /games/yourapikey and thus the data you sent gets saved under your unique key. Then to get it, you would create GET request to the following URL: /games/yourapikey


Data

All HTTP requests contain a header and a body. Header is where you will define what sort of data you will be sending, and the body is where you put your actual data in the format defined by the header. To make sure everything goes fluently, you should define your HTTP request header with "Content-Type" equal to "application/json"

The body should then contain valid JSON-data. Game-data JSON-object should always have the following structure:

{ "game": "NameOfMyGame", "data": { "some": "data" } }

The "data" object can contain whatever valid JSON key-value pairs you want. It can probably support hundreds of paris just fine, but dont go crazy with the size of single inserts.


Route information

Routes Functionality Implemented
/ GET home page Yes
/users GET all the users, POST new user Yes
/users/:apiKey GET user by apiKey, PATCH and PUT by apiKey Yes
/games GET all data about all games Yes
/games/single/:id GET single game data by id Yes
/games/:apiKey GET all games by apiKey, POST new game data Yes
